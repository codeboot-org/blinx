Help on BLINX

1. Introduction

The BLINX wireless device is an accessory to codeBoot, a free programming environment that runs in the browser specifically designed to make programming more accessible and user-friendly. An intuitive user interface avoids the complexities of traditional programming environments. When BLINX is used with codeBoot, it is easy to create programs that interface to the real world. Users can implement functionalities like ambient temperature monitoring, soil humidity tracking, object distance measurement, and more.

2. BLINX device operation

The BLINX device is powered by plugging it into the USB port of a computer, or a 5 watt USB charger, or a USB battery. When it is first powered on, or after the reset button is pressed, the screen will indicate the device's name and the name of the wifi it is trying to connect to, as shown in the following picture:


With factory settings the BLINX device tries to connect to the wifi network named "BLINXNET" with password "blinxnet". This can be changed if desired using this procedure. The red light will blink as long as the connection is not established, in particular if the access-point is out of range or it has reached its capacity (typical access-points have a limit of 20 to 50 connected devices).

To allow wireless communication between codeBoot and BLINX, the computer that is running codeBoot must be connected on the same wifi network as the BLINX device. This network can be isolated from the internet. A typical setup is shown in the following picture:

3. Programmatic acces to BLINX

The BLINX device automatically reads its sensors every second and stores the recent readings in the ".csv" format. These data can then be retrieved by a program as if it were a regular ".csv" file, but at an HTTP path that indicates the address of the BLINX device and the sensors being accessed. For example, this Python code could be executed in codeBoot to read the recent ambient temperature and humidity from BLINX218, based on time intervals of one second, and store them in the variable th :

th = read_file("http://BLINX218.local/temp,humid.csv")

The data can then be viewed in the form of a graph using chart(th) or as a spreadsheet in the editing area by executing write_file("th.csv", th);show_file("th.csv") .

Aide sur BLINX

1. Introduction

L'appareil sans fil BLINX est un accessoire de codeBoot, un environnement de programmation gratuit qui s'exécute dans le navigateur et qui a été spécialement conçu pour rendre la programmation plus accessible et conviviale. Une interface utilisateur intuitive évite les complexités des environnements de programmation traditionnels. Lorsque BLINX est utilisé avec codeBoot, il est facile de créer des programmes qui interagissent avec le monde réel. Les utilisateurs peuvent mettre en œuvre des fonctionnalités telles que la surveillance de la température ambiante, le suivi de l'humidité du sol, la mesure de la distance des objets, et bien plus encore.

2. Opération de l'appareil BLINX

L'appareil BLINX est alimenté en le branchant sur le port USB d'un ordinateur, ou sur un chargeur USB de 5 watts, ou sur une batterie USB. Lorsqu'il est alimenté pour la première fois, ou après avoir appuyé sur le bouton de réinitialisation, l'écran indiquera le nom du dispositif et le nom du réseau wifi auquel il essaie de se connecter, comme indiqué sur l'image suivante:


Avec les réglages d'usine, le dispositif BLINX tente de se connecter au réseau wifi nommé « BLINXNET » avec mot de passe « blinxnet ». Cela peut être modifié si désiré en utilisant cette procédure. La lumière rouge clignotera tant que la connexion ne sera pas établie, en particulier si le point d'accès est hors de portée ou s'il a atteint sa capacité maximale (les points d'accès typiques ont une limite de 20 à 50 appareils connectés).

Pour permettre la communication sans fil entre codeBoot et BLINX, l'ordinateur exécutant codeBoot doit être connecté au même réseau sans fil que l'appareil BLINX. Ce réseau peut être isolé du réseau internet. Une configuration typique est illustrée sur l'image suivante:

3. Accès programmatique à BLINX

L'appareil BLINX fait automatiquement des lectures de ses capteurs à chaque seconde et mémorise les lectures récentes dans le format « .csv ». Ces données peuvent ensuite être récupérées par un programme comme si c'était un fichier « .csv » normal mais à un chemin d'accès HTTP qui indique l'adresse de l'appareil BLINX et les capteurs qui sont consultés. Par exemple ce code Python pourrait être exécuté dans codeBoot pour lire de l'appareil BLINX218 la température et humidité ambiante récente en fonction du temps à chaque seconde et la stocker dans la variable th :

th = read_file("http://BLINX218.local/temp,humid.csv")

Les données peuvent ensuite être visionnées sous forme de graphique avec chart(th) ou sous forme de tableur dans la zone d'édition en exécutant write_file("th.csv", th);show_file("th.csv") .
